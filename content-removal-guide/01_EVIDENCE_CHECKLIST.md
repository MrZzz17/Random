# Phase 1: Evidence & URL Cataloging Checklist

Use this checklist to systematically find and document every instance of your content online before filing any takedown requests. Thorough documentation makes takedowns faster and more successful.

---

## Step 1: Search for Your Content

Run each of these searches in an **incognito / private browsing** window to avoid personalized results.

### Text Searches

Search each of the following on Google, Bing, Yandex, and DuckDuckGo:

- [ ] Your **stage name / screen name** (e.g., `"YourStageName"`)
- [ ] Your **real name** (only if it has been publicly associated with your content)
- [ ] Any **usernames** you have used on cam platforms
- [ ] Combinations: `"YourStageName" + site name` (e.g., `"YourStageName" Chaturbate recorded`)
- [ ] Common repost terms: `"YourStageName" cam recorded`, `"YourStageName" private show`

### Reverse Image Searches

Upload a clear screenshot or profile photo to each of these services:

- [ ] [Google Images](https://images.google.com/) (click camera icon)
- [ ] [Yandex Images](https://yandex.com/images/) (click camera icon вЂ” often best for adult content)
- [ ] [TinEye](https://tineye.com/)

### Platform-Specific Searches

Search directly within each of these sites using your stage name:

- [ ] Pornhub
- [ ] XVideos / XNXX
- [ ] XHamster
- [ ] SpankBang
- [ ] RecTube / CamWhores / similar aggregators
- [ ] Reddit (search `site:reddit.com "YourStageName"`)
- [ ] Twitter/X
- [ ] Telegram (search public channels)

## Step 2: Document Each Instance

For **every** URL where your content appears, record the following in the tracking spreadsheet (see `02_TRACKING_SPREADSHEET.csv`):

| Field | What to Record |
|---|---|
| **URL** | The full URL of the page (copy from browser address bar) |
| **Platform / Site** | The name of the hosting site (e.g., XVideos, Reddit) |
| **Content Type** | Image, Video, or Gallery |
| **Duration** | Length if video (approximate is fine) |
| **Description** | Brief description (e.g., "recorded private show from Chaturbate, Feb 2025") |
| **Date Found** | Today's date |
| **Screenshot Saved?** | Yes / No |
| **Takedown Status** | Not yet filed / Filed / Removed / Escalated |

### How to Take Proper Screenshots

1. Open the page in your browser
2. Make sure the **full URL is visible** in the address bar
3. Include the **date and time** вЂ” you can use your OS clock or an extension like "Webpage Screenshot"
4. Save the screenshot with a descriptive filename: `[site]_[number].png` (e.g., `xvideos_001.png`)
5. Store all screenshots in a single folder for easy reference

> **Tip**: On Mac, press `Cmd+Shift+3` for full screen or `Cmd+Shift+4` to select an area. On Windows, use `Win+Shift+S`.

## Tips

- **Be thorough now** вЂ” finding everything upfront makes the rest of the process much faster
- **Check back periodically** вЂ” content may be re-uploaded to the same or different sites
- **Don't interact with the content** (comment, like, etc.) вЂ” just document and move on
- **Save pages offline** if you are worried they will disappear before you can file (use `Ctrl+S` / `Cmd+S` to save the webpage)
